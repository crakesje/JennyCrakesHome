Module 10: Separation of Content from Presentation

Thus far, I think I have been clearly separating content (HTML) from presentation (CSS) as I create my portfolio website pages within CodePen. I have avoided placing design information within the HTML markup code. The Codecademy exercises were helpful to me in learning what elements belonged within HTML and what belonged within CSS.

On each CodePen page, in the HTML section, I have first created a set of opening and closing tags, and then created a head and body for my document. Within the head, I have my title for the page. In the body, I divide information into categories and place it under headers. I have generally used an h2 size header to list the topic for the page (such as “About”), and an h5 size header for smaller subheads (for example, the title of each project listed on a portfolio subpage.) I have used "div"s to divide my body content into sections that will facilitate styling, such as floating and alignment, later on. On my resume page, I used classes in my html to help me apply uniform styling to sets of HTML elements (for example, dates.)

In the CSS section, I have been sure to follow the general CSS syntax (explained by Codecademy) of:

selector {

property: value;

}

where a selector can be any HTML element, and a property is an aspect of a selector. On every page, I have used the universal selector to set a light blue background color. I always use six-digit HCX codes for my colors. I have also coded for properties such as font-family (using Lucida calligraphy and Helvetica), font-size, and text-align (to center text elements.) Where I had used classes in my html, I applied CSS styling by listing the class as a selector with its own set of brackets.

Moving forward, I plan to maintain separation of content from presentation while I learn to apply more complex CSS design to my document. As I add images, I will code for them within my HTML. The most challenging element I still need to add in, which will involve both HTML and CSS code, is a navigation bar. I will need to float content and link my pages to one another. For clean markup, I can make sure I close off comments with the correct tags.
